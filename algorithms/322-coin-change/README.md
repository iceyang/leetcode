# 322. 零钱兑换

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

示例 1:

```
输入: coins = [1, 2, 5], amount = 11
输出: 3 
解释: 11 = 5 + 5 + 1
```

示例 2:

```
输入: coins = [2], amount = 3
输出: -1
```

说明:
你可以认为每种硬币的数量是无限的。

## 解题思路

题目符合动态规划的特征，属于多阶段决策问题：

* 重复子问题
* 最优子结构
* 无后效性

因此可以采用动态规划来解决。（还有其他解法，比如回溯算法）

这里解题代码里面展示的是两种动态规划，分别是：

* 自下向上（非递归解法）
* 自上向下（递归解法）
